## Starting the Daemon

zlaunch uses a daemon architecture—this enables instant response times and continuous clipboard history monitoring. Start it with:

```bash
zlaunch
```

The daemon runs in the background and creates a socket at `$XDG_RUNTIME_DIR/zlaunch.sock` (typically `/run/user/<uid>/zlaunch.sock`).

<Callout type="info">
To stop the daemon: `zlaunch quit`
</Callout>

## Autostart

Add zlaunch to your compositor's autostart:

<Tabs items={['Hyprland', 'Niri']}>
<Tab value="Hyprland">

Add to `~/.config/hypr/hyprland.conf`:

```bash
exec-once = zlaunch
```

</Tab>
<Tab value="Niri">

Add to `~/.config/niri/config.kdl`:

```kdl
spawn-at-startup "zlaunch"
```

</Tab>
</Tabs>

**Home Manager:**

The Home Manager module exposes an option to autostart zlaunch using a systemd user service.
By default, the service is started at `graphical-session.target`, but this can be configured to start at your compositor's session target, such as `hyprland-session.target`.

```nix
{
  imports = [
    zlaunch.homeManagerModules.default
  ];

  services.zlaunch = {
    enable = true;

    systemd = {
      enable = true;

      autoStart = true; # default
      target = "graphical-session"; # default
    };
  };
}
```


## Keybind Setup

Use `zlaunch toggle` to show/hide the launcher.

<Tabs items={['Hyprland', 'Niri', 'KDE Plasma']}>
<Tab value="Hyprland">

```bash
bind = SUPER, Space, exec, zlaunch toggle
```

</Tab>
<Tab value="Niri">

```kdl
binds {
    Mod+Space { spawn "zlaunch" "toggle"; }
}
```

</Tab>
<Tab value="KDE Plasma">

1. Open **System Settings** → **Shortcuts** → **Custom Shortcuts**
2. Create a new shortcut
3. Set command to `zlaunch toggle`
4. Assign your preferred key

</Tab>
</Tabs>
